Migrando dados para o IBM Cloud

É possível carregar dados de um arquivo de dados em um formato delimitado (CSV ou TXT) localizado em uma rede local ou em um armazenamento de objeto (Amazon S3 ou IBM Cloud Object Storage) para o IBM® Db2® on Cloud. É possível até mesmo migrar seus dados de um sistema no local.

Carregando dados do armazenamento de objeto

Para carregar dados do Amazon S3, selecione um dos métodos a seguir:

  • Console da web do Db2 on Cloud. Carregar > Amazon S3.

  • Tabelas externas diretamente. A seguinte é uma instrução SQL de exemplo:

    INSERT INTO <table-name> SELECT * FROM EXTERNAL '<mys3file.txt>' USING
    (CCSID 1208 s3('s3.amazonaws.com', 
    '<S3-access-key-ID>',
    '<S3-secret-access-key>', 
    '<my_bucket>'
       )
  )

Para carregar dados do IBM Cloud Object Storage usando tabelas externas diretamente, o trecho a seguir é uma instrução SQL de exemplo:

INSERT INTO <table-name> SELECT * FROM EXTERNAL '<mys3file.txt>' USING
  (CCSID 1208 s3('s3-api.us-geo.objectstorage.softlayer.net',
  '<S3-access-key-ID>',
  '<S3-secret-access-key>',
  '<my_bucket>'
     )
  )

Para o IBM Cloud Object Storage, para criar credenciais HMAC ao criar novas credenciais de serviço, especifique {"HMAC:true"} no campo Incluir parâmetros de configuração sequenciais.

Migrando dados do sistema no local

Para migrar seus dados de um sistema no local, escolha um dos métodos a seguir, dependendo do tamanho do seu conjunto de dados:

  • Menor que 25 TB de dados: CLI do IBM Lift
  • 25 TB de dados e maiores: IBM Cloud Mass Data Migration

Lift CLI

A CLI do Lift é um aplicativo que pode ser usado sem encargos para migrar seus dados para o IBM Cloud das várias origens de dados listadas na Tabela 1.

IBM Cloud Mass Data Migration

Uma maneira rápida, simples e segura para transferir fisicamente terabytes a petabytes de dados para o IBM Cloud. O Mass Data Migration é um dispositivo de armazenamento móvel com 120 TB de capacidade de armazenamento utilizável para acelerar a movimentação de dados para o IBM Cloud. Supere desafios comuns de transferência, tais como altos custos, longos tempos de transferência e preocupações com segurança - tudo em um único serviço.

Para obter mais informações sobre o dispositivo Mass Data Migration, veja: Tutorial de introdução.
