Auditoria, criação de log e monitoramento

Auditando

É possível criar uma política de auditoria de banco de dados que fornece a capacidade de auditar várias categorias de eventos que são armazenados em tabelas de eventos de auditoria em seu banco de dados. Para obter mais informações, consulte Diretrizes de política de auditoria .

Também é possível auditar e rastrear mudanças em seu banco de dados usando os métodos a seguir:

  • Ao criar um monitor de eventos CHANGE HISTORY, é possível consultar a tabela de monitor de eventos para determinar o que foi feito dentro do banco de dados e quem fez isso. Para obter mais informações, consulte Monitor de eventos CHANGE HISTORY .
  • O Time Travel Query facilita o armazenamento de todas as mudanças em seus dados e até mesmo consulta seus dados antigos com base em um momento selecionado. Para usar esse método de auditoria, assegure-se de primeiro configurar suas tabelas para suportar o Time Travel Query. Para obter mais informações, consulte Time Travel Query

Para obter mais informações sobre como auditar e rastrear as mudanças do banco de dados, consulte Como auditar ou rastrear mudanças? .

Criando log e monitorando

O monitoramento e a criação de log fazem parte do serviço, no entanto, não são expostos diretamente para o usuário final. Em vez disso, a infraestrutura é usada pela equipe operacional da IBM para tratar de problemas.

Para ver a disponibilidade do Activity Tracker, consulte o roteiro .

É possível se conectar a um cliente da linha de comando do Db2, como o CLPPlus, para acessar informações detalhadas e diagnósticos.

Uma visão geral básica:

Há dois tipos básicos de verificação. Verificações externas de funcionamento/tempo de atividade e monitoramento baseado em métricas. A IBM monitora centenas de métricas e as armazena historicamente. Com base nos valores dessas métricas, os alertas são gerados. Esses alertas são enviados para a equipe de operações da IBM para assegurar que eles sejam vistos e abordados. Além disso, a IBM também envia os logs de diagnósticos e do sistema operacional arquivados para diagnóstico rápido. O Db2 on Cloud está em conformidade com o GDPR e as opções do IBM EU Cloud fornecem um nível ainda mais alto de comprometimento com o GDPR, se necessário.
