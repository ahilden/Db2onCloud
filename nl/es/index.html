Cómo empezar

IBM® Db2® on Cloud es una base de datos SQL suministrada para usted en la nube. Puede utilizar Db2 on Cloud como utilizaría cualquier software de base de datos, pero sin la sobrecarga de la configuración del hardware ni de la instalación y el mantenimiento del software.

También puede instalar una base de datos Db2 local utilizando la descarga gratuita de Db2 Developer Edition . De esta forma, se instala rápidamente una edición de Db2 para desarrolladores lista para utilizar dentro de un contenedor Docker (no se requiere Docker; todos los componentes necesarios se instalarán automáticamente).

Interfaces

Puede trabajar con la base de datos de Db2 on Cloud de los siguientes modos:

  • Consola web de Db2 on Cloud
  • API REST
  • Conecte aplicaciones o herramientas preferidas desde el sistema local
  • Utilice Db2 on Cloud como origen de datos para apps o servicios de IBM Cloud

Consola web de Db2 on Cloud

La consola web proporciona una interfaz gráfica para todo lo que puede utilizar su base de datos, incluyendo: recursos de cargas, un editor SQL, descargas de controladores, etc.

Puede acceder a la consola web de Db2 on Cloud de los siguientes modos:

  • Desde el panel de control de IBM Cloud - Puede abrir la consola web desde la página Detalles de servicio para el servicio de IBM Db2 on Cloud.
  • URL directo - Puede marcar como favorito el URL de la consola web para el servicio de Db2 on Cloud.

Conecte aplicaciones o herramientas preferidas desde el sistema local

Configure el entorno local para conectar la base de datos de Db2 on Cloud siguiendo estos pasos:

  1. Descargue el paquete de controladores de la página Información de conexión de la consola web de Db2 on Cloud.
  2. Instale el paquete de controladores en el sistema donde esté ejecutando las apps o las herramientas.
  3. Configure los archivos de controladores para la base de datos de Db2 on Cloud.

Utilice Db2 on Cloud como origen de datos para apps o servicios de IBM Cloud

Las apps alojadas en IBM Cloud pueden conectarse a la base de datos de Db2 on Cloud exactamente del mismo modo que pueden conectarse las aplicaciones locales a la base de datos de Db2 on Cloud.

Cuando las apps utilizan la plataforma de IBM Cloud, puede sacar partido de la variable de entorno VCAP _SERVICES para simplificar la tarea de especificar detalles y credenciales de bases de datos:

  1. En el panel de control de IBM Cloud, en el separador Conexiones de la página Detalles del servicio para el servicio de IBM Db2 on Cloud, pulse el botón Crear conexión.
  2. Seleccione la app de IBM Cloud para utilizarla con la base de datos de Db2 on Cloud como origen de datos y luego pulse el botón Conectar.

  3. Actualice el código de aplicación para recuperar detalles y credenciales de la base de datos desde la variable de entorno VCAP_SERVICES:

    Ejemplo sin VCAP_SERVICES

     <?php
 $driver      = "DRIVER={IBM DB2 ODBC DRIVER};";

 $database    = "BLUDB";         # Obtenga los detalles de la base de datos desde
 $hostname    = "<Host-name>";   # la página Conectar de
 $port        = 50000;           # la consola web de Db2 on Cloud.
 $user        = "<User-ID >";    #
 $password    = "<Password>";    #
 $dsn         = "DATABASE=$database;" .
                "HOSTNAME=$hostname;" .
                "PORT=$port;" .
                "PROTOCOL=TCPIP;" .
                "UID=$user;" .
                "PWD=$password;";

 $conn_string = $driver . $dsn;

 $conn        = db2_connect( $conn_string, "", "" );
 ?>

    Ejemplo con VCAP_SERVICES

     <?php
 $driver      = "DRIVER={IBM DB2 ODBC DRIVER};";

 $vcap        = json_decode( getenv( "VCAP_SERVICES" ), true );
 $dsn         = $vcap[ "dashDB" ][0][ "credentials" ][ "dsn" ];

 $conn_string = $driver . $dsn;

 $conn        = db2_connect( $conn_string, "", "" );
 ?>
