Lernprogramm 'Einführung'

Bei IBM® Db2® on Cloud handelt es sich um eine SQL-Datenbank, die für Sie in der Cloud bereitgestellt wird. Sie können Db2 on Cloud wie jede andere Datenbanksoftware verwenden, den Zeitaufwand und die Kosten für die Hardwareeinrichtung sowie die Softwareinstallation und -verwaltung fallen jedoch weg.

Erstellen Sie Berechtigungsnachweise. Für die neuen Benutzer von IBM Cloud: Nach der Erstellung des Service müssen Sie einen Benutzernamen und ein Kennwort erstellen, indem Sie beim Starten des Service auf die Schaltfläche Berechtigungsnachweise erstellen klicken. Es ist technisch möglich, sich bei der Webkonsole ohne Berechtigungsnachweise anzumelden, Sie benötigen jedoch den Benutzernamen und das Kennwort zur Verwendung zahlreicher Db2-Tools.

Es ist ebenfalls möglich, eine lokale Db2-Datenbank mithilfe des kostenfreien Downloads für Db2 Developer Edition zu installieren. Dabei wird ohne großen Zeitaufwand eine sofort einsatzfähige Db2 Developer Edition-Instanz mit Tools in einem Docker-Container installiert (Docker nicht erforderlich; alle erforderlichen Komponenten werden automatisch installiert).

Schnittstellen

Für die Arbeit mit der Db2 on Cloud-Datenbank stehen Ihnen die folgenden Möglichkeiten zur Verfügung:

  • Db2 on Cloud-Webkonsole
  • REST-API
  • Herstellen einer Verbindung für Anwendungen oder bevorzugte Tools vom lokalen Computer
  • Verwenden von Db2 on Cloud als Datenquelle für IBM Cloud-Apps oder -Services

Db2 on Cloud-Webkonsole

Die Webkonsole bietet eine grafische Schnittstelle für alle Funktionen, die Sie für die Verwendung der Datenbank benötigen. Hierzu gehören z. B. Ladefunktionen, ein SQL-Editor, Treiberdownloads usw.

Für den Zugriff auf die Db2 on Cloud-Webkonsole stehen Ihnen die folgenden Möglichkeiten zur Verfügung:

  • Über das IBM Cloud-Dashboard - Sie können die Webkonsole über die Servicedetailseite für Ihren IBM Db2 on Cloud-Service öffnen.
  • Direkt über die URL - Sie können die URL der Webkonsole für Ihren Db2 on Cloud-Service mit einem Lesezeichen markieren.

Db2-Befehlszeilenclients und -treiber auf dem Computer installieren

In den meisten Fällen verwenden Benutzer bevorzugt die REST-API oder installieren Treiber für ein Framework, z. B. mit dem Python-Befehl pip.

Zur Installation von Python-Treiberpaketen für MacOS müssen Sie den Befehl pip mit der Option --no-cache-dir verwenden. Ausführliche Anweisungen finden Sie in Python support for IBM DB2 and IBM Informix

Sofern Sie bereits Frameworktreiber installiert haben, können Sie die folgenden Schritte überspringen. Professionelle Anwender möchten möglicherweise den Db2-Befehlszeilenclient verwenden, um ihre Datenbanken zu verwalten und Db2-Befehle zu verwenden. Darüber hinaus kann die Installation von Db2-Treibern grundsätzlich für bestimmte ODBC- oder JDBC-Anwendungen von Vorteil sein. Führen Sie in diesem Fall die folgenden Schritte aus:

  1. Installieren Sie das Treiberpaket auf dem Computer, auf dem Ihre Apps bzw. Tools ausgeführt werden.
  2. Konfigurieren Sie die Treiberdateien für Ihre Db2 on Cloud-Datenbank.

Verwenden von Db2 on Cloud als Datenquelle für IBM Cloud-Apps oder -Services

Für Apps, die per Hosting in IBM Cloud bereitgestellt werden, können Verbindungen zu Ihrer Db2 on Cloud-Datenbank auf dieselbe Weise hergestellt werden wie bei Verbindungen für lokalen Anwendungen zu Ihrer Db2 on Cloud-Datenbank.

Wenn Ihre Apps die IBM Cloud-Plattform verwenden, können Sie die Umgebungsvariable VCAP _SERVICES nutzen, um das Angeben von Datenbankdetails und -berechtigungsnachweisen zu vereinfachen:

  1. Klicken Sie im IBM Cloud-Dashboard auf der Registerkarte Verbindungen der Servicedetailseite für Ihren IBM Db2 on Cloud-Service auf die Schaltfläche Verbindung erstellen.
  2. Wählen Sie die IBM Cloud-App aus, die mit der Db2 on Cloud-Datenbank als Datenquelle verwendet werden soll, und klicken Sie dann auf die Schaltfläche Verbinden.

  3. Aktualisieren Sie den Anwendungscode, um Datenbankdetails- und -berechtigungsnachweise aus der Umgebungsvariablen VCAP_SERVICES abzurufen.

    Beispiel ohne VCAP_SERVICES

     <?php
 $driver      = "DRIVER={IBM DB2 ODBC DRIVER};";

 $database    = "BLUDB";         # Datenbankdetails von der
 $hostname    = "<Host-name>";   # Verbindungsseite der
 $port        = 50000;           # Db2 on Cloud-Webkonsole abrufen.
 $user        = "<Benutzer-ID >";    #
 $password    = "<Kennwort>";    #
 $dsn         = "DATABASE=$database;" .
                "HOSTNAME=$hostname;" .
                "PORT=$port;" .
                "PROTOCOL=TCPIP;" .
                "UID=$user;" .
                "PWD=$password;";

 $conn_string = $driver . $dsn;

 $conn        = db2_connect( $conn_string, "", "" );
 ?>

    Beispiel mit VCAP_SERVICES

     <?php
 $driver      = "DRIVER={IBM DB2 ODBC DRIVER};";

 $vcap        = json_decode( getenv( "VCAP_SERVICES" ), true );
 $dsn         = $vcap[ "dashDB" ][0][ "credentials" ][ "dsn" ];

 $conn_string = $driver . $dsn;

 $conn        = db2_connect( $conn_string, "", "" );
 ?>
