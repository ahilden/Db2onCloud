保守と通知に関するポリシー

IBM® Db2® on Cloud サービスの保守と通知に関するポ リシーは、 IBM Cloud サービスの説明 でご確認いただけます。 Db2 on Cloud に関するサービスの追加条件については、 有料追加 SD カテゴリーでご確認いただけます。

弊社のポリシーを知っておくと、システム保守に対する作業の計画やスケジュール作成 に役立ちます。

保守ポリシー

(2018 年 3 月 27 日から有効) Db2 on Cloud サービスで予定 されているシステム保守を実行する必要がある場合、主要なデータ・センター (ロンドン、ダラス、シドニー ) は別々の曜日の営業日に更新されます。 それ以外のすべてのデータ・センターは、主要データ・センターと重複しない曜日 に一斉に更新されます。 Db2 on Cloud システム保守スケジュールの例 を表 1 に示します。

表 1. システム保守スケジュールの例
データ・センター システム保守スケジュール
ロンドン 月曜
ダラス 火曜
シドニー 水曜
その他すべて 木曜

通知ポリシー

(2018 年 3 月 27 日から有効) お使いの Db2 on Cloud サービスに影響する可能性のあるシステム保守につい ては、予定 日の 2 営業日前にユーザーに通知されます。

緊急事態で時間的な余裕がないと思われるセキュリティー上の脆弱性に 直ちに対処する必要がある場合、 Db2 on Cloud 通知ポリシーは適用されません。 そのような場合は、 IBM Cloud サービスの説明に明記されている通り、少なくとも 24 時間 前に通知が送付されます。
