IBM Cloud へのデータ・マイグレーション

ローカル・ネットワーク上またはオブジェクト・ストア内 (Amazon S3 または IBM Cloud Object Storage) にある、区切り文字で区切られた形式 (CSV または TXT) のデータ・ファイルから IBM® Db2® on Cloud にデータをロードできます。 オンプレミス・システムからデータをマ イグレーションすることもできます。

オブジェクト・ストアからのデータのロード

Amazon S3 からデータをロードするには、以下のいずれかの方法を選択します。

  • Db2 on Cloud Web コンソール。 「ロード (Load)」 > 「Amazon S3」。

  • 外部のテーブルから直接。 次に、SQL ステートメントの例を示します。

    INSERT INTO <table-name> SELECT * FROM EXTERNAL '<mys3file.txt>' USING
  (CCSID 1208 s3('s3.amazonaws.com', 
  '<S3-access-key-ID>',
  '<S3-secret-access-key>', 
  '<my_bucket>'
     )
  )

外部表を使用して、IBM Cloud Object Storage から直接データをロードする場合、以下のサンプル SQL ステートメントを使用できます。

INSERT INTO <table-name> SELECT * FROM EXTERNAL '<mys3file.txt>' USING
  (CCSID 1208 s3('s3-api.us-geo.objectstorage.softlayer.net', 
  '<S3-access-key-ID>',
  '<S3-secret-access-key>', 
  '<my_bucket>'
     )
  )

注: IBM Cloud Object Storage の場合、HMAC 資格情報を作成するために、 新規サービス資格情報を作成する際、 「インラインの構成パラメーターの追加」フィールド に {"HMAC:true"} を指定してください。

オンプレミス・システムからのデータのマイグレーション

オンプレミス・システムからデータをマイグレーションするには、データ・セットのサイズに応じて 、以下のいずれかの方式を選択します。

  • 25 TB 未満のデータ: IBM Lift CLI
  • 25 TB 以上のデータ: IBM Cloud Mass Data Migration

Lift CLI

Lift CLIは、表 1 にリストされている各種データ・ソースから IBM Cloud にデータをマイグレーションするときに無料で使用できるア プリケーションです。

表 1. マイグレーションのデータ・ソース
IBM Cloud 上のターゲット DB データ・ソース
IBM Db2 on Cloud IBM Db2
Oracle データベース
Microsoft SQL Server
CSV ファイル・フォーマット

Lift CLI のダウンロードとインストールについては、 Lift CLI のダウンロード を参照してください。

Lift CLI を使用して IBM Cloud にデータをマイグレーションする手順 については、 IBM Db2 on Cloud へのデータ・マイグレーション を参照してください。

IBM Cloud Mass Data Migration

テラバイト単位からペタバイト単位までのデータを IBM Cloud に物理的に転送するための高速でシンプルで安全な方法です。 Mass Data Migrationは 120 TB の使用可能な記憶容量を備えたモ バイル・ストレージ・デバイスで、IBM Cloud へのデータ移動を高速化す ることが できます。 高いコスト、長い転送時間、セキュリティー上の懸念など、転送に関する共通の な課題を単一のサービスですべて克服できます。

Mass Data Migration デバイスについて詳しくは、 IBM Cloud Mass Data Migration 入門 を参照してください。
