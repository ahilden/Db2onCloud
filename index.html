Getting started

IBM® Db2® on Cloud is an SQL database provisioned for you in the cloud. You can use Db2 on Cloud just as you would use any database software, but without the overhead and expense of hardware setup or software installation and maintenance.

You can also install a local Db2 database using the free Db2 Developer Edition download . It rapidly installs a ready-to-go developer edition of Db2 with tools inside a Docker container (Docker not required; it will automatically install any necessary components).

Interfaces

You can work with your Db2 on Cloud database in the following ways:

  • Db2 on Cloud web console
  • REST API
  • Connect applications or your favorite tools from your local computer
  • Use Db2 on Cloud as a data source for your IBM Cloud apps or services

Db2 on Cloud web console

The web console provides a graphical interface for everything that you need to use your database, including: load facilities, an SQL editor, driver downloads, and more.

You can access your Db2 on Cloud web console in the following ways:

  • From your IBM Cloud dashboard - You can open the web console from the Service Details page for your IBM Db2 on Cloud service.
  • Direct URL - You can bookmark the URL of the web console for your Db2 on Cloud service.

Installing Db2 command line clients and drivers on your computer

Most users can skip this step. In most cases, users tend to use only the REST API, or install drivers for a framework, for example, with Python's pip command. However, power users might want to use the Db2 command line client to administer their database and use Db2 commands. Also, certain ODBC or JDBC applications can benefit from a general installation of Db2 drivers. If so, complete the following steps:

  1. Install the driver package on the computer where your apps or tools are running.
  2. Configure the driver files for your Db2 on Cloud database.

Use Db2 on Cloud as a data source for your IBM Cloud apps or services

Apps hosted on IBM Cloud can connect to your Db2 on Cloud database exactly the same way that your local applications connect to your Db2 on Cloud database.

When your apps use the IBM Cloud platform, you can take advantage of the VCAP _SERVICES environment variable to simplify the task of specifying database details and credentials:

  1. On your IBM Cloud dashboard, in the Connections tab of the Service Details page for your IBM Db2 on Cloud service, click the Create connection button.
  2. Select the IBM Cloud app to use with your Db2 on Cloud database as a data source, and then click the Connect button.

  3. Update your application code to retrieve database details and credentials from the VCAP_SERVICES environment variable:

    Example without VCAP_SERVICES

     <?php
 $driver      = "DRIVER={IBM DB2 ODBC DRIVER};";

 $database    = "BLUDB";         # Get these database details from
 $hostname    = "<Host-name>";   # the Connect page of the Db2 on Cloud
 $port        = 50000;           # web console.
 $user        = "<User-ID >";    #
 $password    = "<Password>";    #
 $dsn         = "DATABASE=$database;" .
                "HOSTNAME=$hostname;" .
                "PORT=$port;" .
                "PROTOCOL=TCPIP;" .
                "UID=$user;" .
                "PWD=$password;";

 $conn_string = $driver . $dsn;

 $conn        = db2_connect( $conn_string, "", "" );
 ?>

    Example with VCAP_SERVICES

     <?php
 $driver      = "DRIVER={IBM DB2 ODBC DRIVER};";

 $vcap        = json_decode( getenv( "VCAP_SERVICES" ), true );
 $dsn         = $vcap[ "dashDB" ][0][ "credentials" ][ "dsn" ];

 $conn_string = $driver . $dsn;

 $conn        = db2_connect( $conn_string, "", "" );
 ?>
